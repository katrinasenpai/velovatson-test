# Публикация теста

## Что загружать

Для хостинга используйте содержимое папки `velovatson-test` или архив `velovatson-site.zip`.

В корне сайта должен лежать файл `index.html`, рядом:

- `styles.css`
- `script.js`
- папка `assets`

## GitHub Pages

1. Создайте пустой репозиторий на GitHub.
2. Загрузите содержимое папки `velovatson-test` в корень репозитория.
3. В настройках репозитория откройте `Pages`.
4. Включите публикацию из ветки `main`, папка `/root`.
5. Откройте выданный GitHub Pages URL на флипчарте.

## Timeweb Cloud App Platform

1. Создайте приложение из репозитория.
2. Выберите тип приложения `HTML/CSS/JS`.
3. Укажите директорию со статическими файлами: корень репозитория, если загружали файлы из `velovatson-test` прямо в корень.
4. После деплоя откройте публичный URL приложения на флипчарте.

## Timeweb Cloud Server

Если у вас обычный облачный сервер с nginx/Apache, загрузите содержимое папки `velovatson-test` в директорию сайта, например:

- `/var/www/html`
- или папку сайта в панели управления

Backend и база данных не нужны.
